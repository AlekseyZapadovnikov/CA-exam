# Computer Architecture Exam Prep

Неофициальный репозиторий для подготовки к устному экзамену по курсу
«Архитектура компьютера».

Внутри лежат экзаменационные билеты, короткие версии для повторения и
подготовленные источники по лекциям: слайды, транскрипты, личные заметки и
компактные source-pack-конспекты.

## Что готово

- [tickets/lecture-01.md](tickets/lecture-01.md) — билеты по первой лекции.
- [tickets/lecture-01.cram.md](tickets/lecture-01.cram.md) — краткое повторение по первой лекции.
- [lectures.yaml](lectures.yaml) — карта лекций, источников и статусов подготовки.

Актуальный статус каждой лекции смотри в `lectures.yaml` и в блоке
`Статус подготовки` в конце соответствующего файла `tickets/lecture-XX.md`.

## Как пользоваться

1. Открой нужный файл `tickets/lecture-XX.md`.
2. Начни с раздела `Короткий ответ`: это версия, которую удобно произнести на экзамене.
3. Потом прочитай `Схема / картинка` и `Подробный ответ`.
4. Перед самопроверкой пройди блоки `Возможные вопросы преподавателя` и
   `Что обязательно запомнить`.
5. Если времени мало, используй `tickets/lecture-XX.cram.md`.

Метки в билетах важны:

- `[проверить]` — место требует ручной сверки с исходниками;
- `[нет в источниках]` — в материалах лекции не найдено подтверждение.

## Принципы

- Один экзаменационный вопрос — один билет.
- Билеты пишутся для устного ответа, а не как полный пересказ лекции.
- Факты берутся из материалов конкретной лекции.
- Если утверждение не подтверждается источниками, оно помечается явно.
- Внешние пояснения не смешиваются с фактами лекции.

## Структура репозитория

- `tickets/` — готовые билеты и краткие версии для повторения.
- `sources/lecture-XX/` — подготовленные источники конкретной лекции:
  заметки, слайды в Markdown, транскрипты и `source-pack.md`.
- `slides/lecture-XX/` — исходные материалы слайдов, если они есть отдельно.
- `assets/lecture-XX/` — изображения, полученные при извлечении слайдов.
- `csa-rolling/` — материалы курса и список экзаменационных вопросов, используемые как источник формулировок и слайдов.
- `lectures.yaml` — список лекций, ссылок, путей и статусов.
- `scripts/` — вспомогательные скрипты для транскриптов и слайдов.
- `prompts/` — промпты для очистки, сборки source-pack, генерации и ревью билетов.
- `docs/maintenance.md` — как обновлять и пересобирать материалы.
- `AGENTS.md` и `PIPELINE.md` — служебные инструкции для AI-агента.

## Источники

Главный список экзаменационных вопросов:

- [csa-rolling/exam-questions-blitz.md](csa-rolling/exam-questions-blitz.md)

Для каждого билета источники берутся из соответствующей папки
`sources/lecture-XX/` в таком приоритете:

1. `my-notes.md`
2. `slides.md`
3. `transcript.cleaned.md`
4. `transcript.raw.md`

`source-pack.md` используется как компактная карта лекции, но спорные места
должны сверяться с исходными файлами.

## Обновление материалов

Коротко:

```bash
python3 scripts/fetch_transcript.py --lecture lecture-XX
python3 scripts/extract_slides.py --lecture lecture-XX
```

После этого материалы очищаются, собираются в `source-pack.md`, превращаются в
билет, проходят ревью и финализацию. Подробный порядок описан в
[docs/maintenance.md](docs/maintenance.md).

Если работа выполняется AI-агентом, он должен следовать [AGENTS.md](AGENTS.md)
и [PIPELINE.md](PIPELINE.md).

## Ограничения

- Это неофициальные учебные материалы, а не официальный репозиторий курса.
- Автоматические транскрипты могут содержать ошибки распознавания.
- Билет со статусом `needs-check` нельзя считать полностью проверенным.
- Перед публичной публикацией проверь права на исходные материалы курса,
  слайды, изображения, транскрипты и содержимое `csa-rolling/`.
