# Computer Architecture Exam

Проект помогает готовить экзаменационные билеты по архитектуре компьютера на основе лекций с YouTube, слайдов, транскриптов и личных заметок.

Главный принцип: итоговые ответы должны опираться на материалы лекций. Если информации нет в источниках, в билете нужно писать `[нет в источниках]`, а сомнительные места помечать `[проверить]`.

## Структура

- `lectures.yaml` — главный список лекций.
- `sources/lecture-XX/` — материалы конкретной лекции: транскрипт, слайды в Markdown, заметки и компактный `source-pack.md`.
- `slides/lecture-XX/` — исходные PDF-файлы слайдов.
- `assets/lecture-XX/` — изображения страниц слайдов и извлеченные схемы.
- `tickets/lecture-XX.md` — итоговый билет.
- `tickets/lecture-XX.cram.md` — короткая версия для повторения.
- `prompts/` — промпты для этапов обработки.
- `scripts/` — вспомогательные скрипты.
- `logs/` — логи автоматической обработки.
- `PIPELINE.md` — подробный пайплайн для агента: как из номера лекции получить готовые билеты и краткую версию.

## Как пользоваться
1) Переходите в папку репозитория и открываете окно чата с вашим агентом
2) Пишите промпт - "Напиши билеты для лекции X" (где X - это номер лекции)
3) Агент сам скачает субтитры лекции, слайды презентации и сам напишет билет
4) Смотри результат в папке `tickets` файл `lecture-01.md`


## Зависимости

Минимально нужен Python 3.10+.

Опционально:

- `youtube-transcript-api` — для автоматического получения YouTube-транскриптов;
- `PyYAML` — для полноценного чтения YAML;
- `PyMuPDF` (`fitz`) — для извлечения текста и изображений из PDF;
- `pypdf` и `pdftoppm` — возможный fallback для PDF.

## Предупреждения

- Автоматические YouTube-субтитры могут быть неточными.
- Если транскрипт не удалось получить автоматически, пользователь должен вставить его вручную в `sources/lecture-XX/transcript.raw.md`.
- Итоговый билет нельзя считать готовым, пока он не прошел ревью (Смотрите лекции сами, модифицируйте проект под себя).
- Если слайды плохо извлеклись автоматически, проверь картинки в `assets/lecture-XX/` и вручную дополни `sources/lecture-XX/slides.md`.

## Важно
- Репозиторий создан, чтобы автоматизировать процесс НАПИСАНИЯ (ручкой или на клавиатуре) билета, он не создан для того, чтобы быть единственным источником истины для подготовки к экзамену, более того, пока что он почти не протестирован - проверяйте билеты руками.
