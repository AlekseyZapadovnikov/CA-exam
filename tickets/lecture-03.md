## Билет 1. Сравните подходы к реализации вычислений в арифмометре и на логарифмической линейке. Аналогии с современными компьютерами.

### Короткий ответ

Арифмометр и логарифмическая линейка реализуют вычисления разными способами.
Линейка реализует аналоговое вычисление, то есть результат задаётся непрерывным физическим положением шкал, а не точным набором дискретных разрядов.
Она аналоговая, потому что пользователь выставляет шкалы и считывает значение по отметкам.
Поэтому результат получается быстро, но приближённо.
Точность зависит от шкалы, глаза и навыка пользователя.
А арифмометр реализует цифровое вычисление.
Он работает с дискретными разрядами и выполняет операцию как последовательность шагов.
Одно вычисление на арифмометре может идти дольше, чем поиск по шкале или таблице.
Зато при росте разрядности арифмометр масштабируется лучше, чем полная таблица значений.
Современная аналогия с линейкой и таблицей — таблицы истинности, cache, memoization и SRT division, а с арифмометром — аппаратное сложение и деление по шагам.

## Билет 2. Каков был подход к расчёту артиллерийских таблиц группой людей? Аналогии с современными компьютерами (параллелизм уровня инструкций, надёжность).

### Короткий ответ

#### Каков был подход к расчёту артиллерийских таблиц группой людей?

Артиллерийские таблицы считали как поток вычислений, распределённый между людьми.
Сложную формулу представляли в виде графа операций.
Каждую вершину графа поручали человеку, который выполнял одну простую операцию.
Человек знал, от кого получить входные данные и кому передать результат.
На вход графа подавали параметры, а на выходе собирали готовые значения.

#### Какие есть аналогии с современными компьютерами (параллелизм уровня инструкций, надёжность)?

Независимые вершины графа можно было считать одновременно, как параллелизм уровня инструкций.
Новые наборы входных данных запускали следом, что похоже на конвейеризацию.
Надёжность повышали специализацией, потому что каждый узел делал одну операцию.
Её также повышали двоированием: два расчёта сравнивали, а при несовпадении повторяли.
Современные аналоги этого подхода — внутренний параллелизм процессора, вычислительные конвейеры и избыточные вычисления для проверки результата.

## Билет 3. Каково устройство электрического реле? Какие виды реле существуют и каковы области их применения?

### Короткий ответ

#### Каково устройство электрического реле?

Электрическое реле — это управляемый ключ между входом и выходом.
Ключом управляет магнитная катушка: ток притягивает контакт, а пружина возвращает его в нормальное состояние.
Реле работает с уровнями `0` и `1`: цепь либо замкнута, либо разомкнута.

#### Какие виды реле существуют?

Реле бывают электрические, механические, пневматические, тепловые, оптические, акустические и магнитные.
Их общий смысл одинаковый: разными физическими способами заставить ключ замкнуться или разомкнуться.

#### Каковы области применения реле?

Реле исторически использовали для построения компьютеров.
Сейчас они полезны в силовых шкафах управления, где есть большие токи и тяжёлая среда эксплуатации.
Неэлектрические реле нужны там, где обычная электроника работает плохо или не работает.
Ещё один пример применения релейного подхода — программируемые логические контроллеры.

## Билет 4. Как реализуется булев базис на электрических реле?

### Короткий ответ

Булев базис на реле реализуется через способы соединения контактов.
В слайдах разбираются операции `AND`, `OR` и `NOT`.
Для `AND` контакты ставят последовательно.
Ток проходит только если активны оба контакта.
Для `OR` контакты ставят параллельно.
Ток проходит, если активен хотя бы один путь.
Для `NOT` используют нормально замкнутый контакт или инверсный элемент диаграммы.
Такой контакт проводит ток в обычном состоянии и размыкается при активации.
Результат подаётся на актуатор, например мотор двери или другое управляемое устройство.
Поэтому релейная диаграмма превращает логическое выражение в физическую цепь.

### Схема / картинка

```text
 L1                                      L2
 o                                       o
 |             Logical AND               |
 |                                       |
 +--[ ]--------[ ]-----------( )---------+
 |  Key 1      Key 2         Door motor  |
 |                                       |
```

```text
 L1                                      L2
 o                                       o
 |             Logical OR                |
 |                                       |
 +----+--[ ]--------------+--( )---------+
 |    |  Exterior unlock  |  Unlock      |
 |    |                   |              |
 |    +--[ ]--------------+              |
 |        Interior unlock                |
 |                                       |
```

Схемы из слайдов показывают последовательное соединение для `AND` и параллельное соединение для `OR`.

## Билет 5. Опишите классическую схему включения электродвигателя с аварийной остановкой. Объясните, как работает обратная связь.

### Короткий ответ

Классическая схема включения двигателя строится вокруг цепи `Run`.
В начале аварийная кнопка `ES` и кнопка `Stop` находятся в нормально замкнутом состоянии.
Кнопка `Start` нормально разомкнута.
Пока `Start` не нажата, реле `Run` не включено и мотор выключен.
При нажатии `Start` ток проходит через `ES`, `Stop` и `Start` к катушке `Run`.
Реле `Run` срабатывает и замыкает свой контакт параллельно `Start`.
После отпускания `Start` ток продолжает идти через контакт `Run`.
Нижний контакт `Run` включает `Motor`.
Если нажать `Stop` или аварийную кнопку `ES`, цепь разрывается.
Обратная связь состоит в том, что выход `Run` возвращается в управляющую цепь и удерживает новое состояние.

### Схема / картинка

```text
 L1                                        L2
 o                                         o
 |                                         |
 +--[\]---[\]----+--[ ]---+---------( )----+
 |  ES    Stop   |  Start |         Run    |
 |               |        |                |
 |               +--[ ]---+                |
 |                  Run                    |
 |                                         |
 |                                         |
 |                                         |
 +------------------[ ]-------------( )----+
 |                  Run         Motor      |
 |                                         |
```

Это схема из слайдов: `Run` параллельно `Start` делает самоподхват, а `ES` и `Stop` разрывают цепь.
