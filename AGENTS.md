# Правила работы агента

## Язык и формат

- Писать на русском языке.
- Все билеты делать для устного экзамена по архитектуре компьютера.
- Математические формулы писать в аккуратном LaTeX display-формате.
- Итоговые билеты хранить в `tickets/lecture-XX.md`.

## Источники и приоритет

Главные источники, в порядке приоритета:

1. мои заметки `sources/lecture-XX/my-notes.md`;
2. слайды `sources/lecture-XX/slides.md`;
3. очищенный транскрипт `sources/lecture-XX/transcript.cleaned.md`;
4. сырой транскрипт `sources/lecture-XX/transcript.raw.md`.

Для каждого утверждения в билете желательно понимать, из какого файла оно взято.

## Запрет на выдумывание

- Не выдумывать факты.
- Если информации нет в источниках, писать `[нет в источниках]`.
- Если место сомнительное, писать `[проверить]`.
- Внешние знания использовать только для пояснения и явно помечать отдельным блоком `[внешнее пояснение]`.
- Не смешивать факты из разных лекций без явной пометки.

## Работа с материалами

- Не удалять исходные материалы.
- Не переписывать `transcript.raw.md` вручную.
- Не считать билет готовым, пока он не прошел ревью.
- Если автоматический транскрипт или извлечение слайдов выглядят плохо, помечать проблемные места `[проверить]`.
- Если транскрипт лекции не удалось получить автоматически и пользователь не предоставил его вручную, генерацию билетов нужно остановить. В этом случае нельзя создавать или обновлять `tickets/lecture-XX.md` и `tickets/lecture-XX.cram.md`; можно только зафиксировать причину остановки в логе/ответе.

## Билеты

- Перед генерацией, ревью или финализацией билетов обязательно читать и применять `PIPELINE.md`.
- Перед написанием билетов обязательно должен быть настоящий `sources/lecture-XX/transcript.raw.md`, а не заглушка или запись `method: "not-fetched"`. Без транскрипта билет не писать даже по готовым слайдам.
- Билеты писать строго по формулировкам вопросов из `csa-rolling/exam-questions-blitz.md`: один экзаменационный вопрос — один билет/раздел ответа.
- Заголовок билета должен повторять формулировку вопроса по смыслу, лучше дословно; ответ должен закрывать все части вопроса, а не пересказывать лекцию целиком.
- Ответ должен быть пригоден для устного экзамена: короткий ответ, подробное объяснение, определения, пример, возможные вопросы преподавателя и список того, что обязательно запомнить.
- Ответ должен строиться от формулировки вопроса. Например, на вопрос "Как можно рассматривать систему с точки зрения функционального места?" короткий ответ должен начинаться по смыслу так: "Систему с точки зрения функционального места можно рассматривать как ...".
- `Короткий ответ` должен быть максимально простым: 9-10 коротких предложений, без вводных вроде "в первой лекции", без длинных списков и без тяжёлых терминов без пояснения.
- Если лектор приводит примеры в лекции, в `Короткий ответ` желательно включить один самый полезный пример с коротким пояснением, почему он подходит; не нужно перечислять все примеры.
- Если вопрос требует перечисления состава, уровней или этапов, в `Коротком ответе` дать только суть и один самый полезный пример, а полный список и пояснения раскрыть в `Подробном ответе`.
- Блок `Схема / картинка` в каждом билете должен идти сразу после `Короткий ответ` и перед `Подробный ответ`.
- Если в ответе используется схема со списком уровней, этапов или компонентов, в `Подробном ответе` нужно кратко пояснить каждый пункт простыми словами, а не только переписать подписи с картинки.
- Слайды использовать как источник определений, схем, таблиц, формул и картинок.
- Транскрипт использовать как источник объяснений преподавателя.
- `my-notes.md` использовать как самый приоритетный источник.
