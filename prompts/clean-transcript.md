# Очистка транскрипта лекции

Ты очищаешь `sources/lecture-XX/transcript.raw.md` и создаешь `sources/lecture-XX/transcript.cleaned.md`.

## Цель

Сделать транскрипт пригодным для подготовки устного экзамена по архитектуре компьютера, не меняя смысл лекции.

## Источники

Используй только:

- `sources/lecture-XX/transcript.raw.md`;
- при необходимости `sources/lecture-XX/my-notes.md` для расшифровки терминов, но не добавляй новых фактов без пометки.

## Правила

- Не менять смысл.
- Не добавлять факты, которых нет в транскрипте.
- Убрать мусор авто-субтитров: повторы, обрывы, очевидные сбои распознавания, лишние междометия.
- Сохранить технические термины.
- Сохранить важные численные значения, формулы, названия команд, регистров, блоков, устройств и архитектур.
- Разбить текст на логические разделы с короткими заголовками.
- Непонятные места помечать `[проверить]`.
- Если термин или фраза восстановлены предположительно, пометить `[проверить]`.
- Если в источнике явно нет данных, не дополнять внешними знаниями.

## Выходной формат

```markdown
# Lecture XX — очищенный транскрипт

## Метаданные

- lecture id: `lecture-XX`
- источник: `transcript.raw.md`
- статус: `cleaned-draft`

## Раздел 1. Название

Очищенный текст.

## Раздел 2. Название

Очищенный текст.

## Непонятные места

- `[проверить]` ...

## Потенциальные экзаменационные темы

- ...
- ...
- ...
```

## Самопроверка

Перед сохранением проверь:

- не появился ли новый смысл;
- не исчезли ли определения, примеры и формулы;
- все ли сомнительные места помечены `[проверить]`;
- можно ли по очищенному тексту собрать `source-pack.md`.
