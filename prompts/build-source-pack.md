# Сборка `source-pack.md`

Ты собираешь `sources/lecture-XX/source-pack.md`.

`source-pack.md` — главный компактный источник для генерации билета. Он должен быть короче исходных материалов, но сохранять все важное для устного экзамена по архитектуре компьютера.

## Приоритет источников

1. `sources/lecture-XX/my-notes.md`
2. `sources/lecture-XX/slides.md`
3. `sources/lecture-XX/transcript.cleaned.md`
4. `sources/lecture-XX/transcript.raw.md`

Каноническое место просмотра слайдов — `https://csa.edu.swampbuds.me/`.
Перед сборкой source-pack убедись, что `slides.md` относится к deck-у текущей
лекции. Если локальные слайды выглядят как материал соседней лекции или не
совпадают с ожидаемой темой, пометь это `[проверить]`.

Для раздела `Схемы и картинки` ориентируйся на канонический рендер слайдов на
`https://csa.edu.swampbuds.me/`. Если цельный слайд важен сам по себе
(заголовок, список, схема, подписи и расположение элементов вместе), фиксируй
именно цельный слайд и путь к его скриншоту/рендеру, а не только отдельную
внутреннюю картинку из локального markdown.

Если источники противоречат друг другу, укажи противоречие и пометь `[проверить]`.

## Правила

- Не выдумывать факты.
- Не добавлять внешние знания без отдельного блока `[внешнее пояснение]`.
- Не смешивать факты из разных лекций без пометки и явной причины.
- Не использовать слайды соседней лекции для закрытия пробелов текущей лекции
  без отдельного решения пользователя.
- Для важных утверждений указывать источник в скобках: `(my-notes)`, `(slides)`, `(transcript.cleaned)`, `(transcript.raw)`.
- Если информации нет, писать `[нет в источниках]`.
- Если место сомнительное, писать `[проверить]`.

## Самопроверка

Перед сохранением проверь:

- по `source-pack.md` реально можно сгенерировать билет;
- каждый важный факт связан с источником;
- все пробелы явно помечены;
- не потеряны темы, повторяющиеся в слайдах и транскрипте.
