# Финализация экзаменационного билета

Ты финализируешь `tickets/lecture-XX.md` после ревью.

## Источники

- текущий `tickets/lecture-XX.md`;
- ревью билета;
- `sources/lecture-XX/source-pack.md`;
- `sources/lecture-XX/my-notes.md`;
- `sources/lecture-XX/slides.md`;
- `sources/lecture-XX/transcript.cleaned.md`;
- `sources/lecture-XX/transcript.raw.md`.

## Правила

- Исправить ошибки из ревью.
- Убрать неподтвержденные утверждения.
- Добавить пропущенные темы, если они есть в источниках.
- Не переписывать весь билет без необходимости.
- Сохранить стиль устного ответа.
- Не добавлять факты без источников.
- Если информацию нельзя подтвердить, оставить `[нет в источниках]`.
- Если место нужно проверить человеком, оставить `[проверить]`.
- Картинки оставлять только там, где они помогают ответу.
- В каждом билете блок `Схема / картинка` должен идти сразу после `Короткий ответ` и перед `Подробный ответ`.
- Переписать перегруженные `Короткие ответы`: оставить 9-10 простых коротких предложений, убрать лишние вводные вроде "в первой лекции" и длинные перечисления.
- Проверить, что `Короткий ответ` строится от формулировки вопроса: например, вопрос "Как можно рассматривать систему..." должен получать ответ "Систему можно рассматривать как ...".
- Если лектор приводит примеры в лекции, в `Короткий ответ` желательно включить один самый полезный пример с коротким пояснением, почему он подходит; не нужно перечислять все примеры.
- Если список уровней, компонентов или этапов убран из короткого ответа, раскрыть его в `Подробном ответе` простыми пояснениями.

## Что обновить

- `tickets/lecture-XX.md` — финальная версия билета.
- `tickets/lecture-XX.cram.md` — короткая версия для повторения.

## Формат статуса в конце `tickets/lecture-XX.md`

```markdown
---

## Статус подготовки

- статус: `reviewed` / `needs-check`
- дата финализации: `YYYY-MM-DD`
- оставшиеся проверки:
  - ...
```

## Формат `tickets/lecture-XX.cram.md`

```markdown
# Lecture XX — краткое повторение

## Главное

- ...
- ...
- ...

## Определения

- **Термин** — определение.

## Формулы

$$
...
$$

## Типовые вопросы

- ...

## Нужно проверить

- `[проверить]` ...
- `[нет в источниках]` ...
```

## Самопроверка

Перед завершением проверь:

- билет исправлен по ревью;
- краткая версия не содержит новых фактов;
- неподтвержденные утверждения удалены или помечены;
- картинки стоят сразу после `Короткий ответ`;
- короткие ответы простые и не перегружены перечислениями;
- в конце есть статус подготовки.
