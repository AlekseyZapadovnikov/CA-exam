# Финализация экзаменационного билета

Ты финализируешь `tickets/lecture-XX.md` после ревью.

Главная цель финализации — получить короткий, плотный билет без раздела
`Подробный ответ`.

## Источники

- текущий `tickets/lecture-XX.md`;
- ревью билета;
- `sources/lecture-XX/source-pack.md`;
- `sources/lecture-XX/my-notes.md`;
- `sources/lecture-XX/slides.md`;
- `sources/lecture-XX/transcript.cleaned.md`;
- `sources/lecture-XX/transcript.raw.md`.

## Правила

- Исправить ошибки из ревью.
- Убрать неподтверждённые утверждения.
- Не добавлять факты без источников.
- Если информацию нельзя подтвердить, оставить `[нет в источниках]`.
- Если место нужно проверить человеком, оставить `[проверить]`.
- **Удалить раздел `Подробный ответ`, если он есть.**
- Не заменять удалённый подробный ответ другим длинным разделом.
- Сжать материал до плотных смысловых блоков.
- В `Коротком ответе` использовать `####`-подзаголовки.
- Ключевые мысли и термины выделять **жирным**.
- Длинные абзацы переписать в короткие маркеры.
- Каждый маркер должен нести одну мысль.
- Блок `Схема / картинка` должен идти сразу после `Короткий ответ`.
- Картинки оставлять только там, где они помогают ответу.
- Не придумывать своё разбиение билета на тематические блоки. Использовать
  вопросы и части, явно заданные в заголовке билета.
- Если в заголовке несколько вопросов, в `Коротком ответе` должны быть
  отдельные блоки по этим вопросам.
- Если какая-то часть формулировки не подтверждается источниками, ответить на
  неё отдельно и поставить `[нет в источниках]` или `[проверить]`.
- Не писать мета-фразы вроде "в лекции", "в слайдах", "источники говорят",
  "лектор говорит".

## Что обновить

- `tickets/lecture-XX.md` — финальная версия билета.
- `tickets/lecture-XX.cram.md` — короткая версия для повторения.

## Финальный формат билета

```markdown
## Билет N. Точная формулировка вопроса

### Короткий ответ

#### Суть

- **...** — ...

#### Части вопроса

- **...** — ...

#### Что важно не перепутать

- **...** — ...

### Схема / картинка

![Описание](../assets/lecture-XX/slide-001.png)

или `[подходящей картинки не найдено]`.

### Опорные определения

- **Термин** — коротко.

### Пример

- **Пример** — коротко, почему подходит.

### Возможные вопросы преподавателя

- ...

### Что обязательно запомнить

- **...**

### Проверить

- `[проверить]` ...
- `[нет в источниках]` ...
```

## Формат статуса в конце `tickets/lecture-XX.md`

```markdown
---

## Статус подготовки

- статус: `reviewed` / `needs-check`
- дата финализации: `YYYY-MM-DD`
- оставшиеся проверки:
  - ...
```

`reviewed` ставить только если все существенные утверждения подтверждены
источниками и нет критичных `[проверить]`.

`needs-check` ставить, если:

- транскрипт автоматический и есть сомнительные места;
- источник противоречив;
- вопрос из экзаменационного списка покрыт лекцией не полностью;
- не хватает данных для уверенного ответа.

## Формат `tickets/lecture-XX.cram.md`

Краткая версия не должна содержать новых фактов.

```markdown
# Lecture XX — краткое повторение

## Главное

- **...**

## Определения

- **Термин** — определение.

## Формулы

$$
...
$$

## Типовые вопросы

- ...

## Нужно проверить

- `[проверить]` ...
- `[нет в источниках]` ...
```

Если формул нет, писать `[формул в источниках не найдено]`.

## Самопроверка

Перед завершением проверь:

- раздела `Подробный ответ` нет;
- билет исправлен по ревью;
- краткая версия не содержит новых фактов;
- неподтверждённые утверждения удалены или помечены;
- `Короткий ответ` разбит на смысловые блоки;
- ключевые мысли выделены **жирным**;
- длинные абзацы сокращены;
- картинки стоят сразу после `Короткий ответ`;
- каждая часть заголовка получила прямой ответ;
- в конце есть статус подготовки.
