# Финализация экзаменационного билета

Ты финализируешь `tickets/lecture-XX.md` после ревью.

Главная цель финализации — получить полноценный устный билет без раздела
`Подробный ответ`. Весь основной материал должен быть в `Коротком ответе`.

## Источники

- текущий `tickets/lecture-XX.md`;
- ревью билета;
- `sources/lecture-XX/source-pack.md`;
- `sources/lecture-XX/my-notes.md`;
- `sources/lecture-XX/slides.md`;
- `sources/lecture-XX/transcript.cleaned.md`;
- `sources/lecture-XX/transcript.raw.md`;
- канонические слайды на `https://csa.edu.swampbuds.me/`, если нужно сверить
  правильный deck.

Картинки финализировать по каноническому рендеру на
`https://csa.edu.swampbuds.me/`. Если билет закрывает цельный слайд, вставлять
скриншот/рендер всего слайда в `assets/lecture-XX/`, а не отдельную внутреннюю
схему из локального markdown.

## Правила

- Исправить ошибки из ревью.
- Убрать неподтверждённые утверждения.
- Не добавлять факты без источников текущей лекции.
- Не брать факты и картинки из соседней лекции без явной пометки и решения
  пользователя.
- Если информацию нельзя подтвердить, оставить `[нет в источниках]`.
- Если место нужно проверить человеком, оставить `[проверить]`.
- **Удалить раздел `Подробный ответ`, если он есть.**
- Не заменять удалённый подробный ответ другим длинным служебным разделом.
- Не возвращать старые секции по умолчанию: `Опорные определения`,
  `Ключевые определения`, `Пример`, `Возможные вопросы преподавателя`,
  `Что обязательно запомнить`, `Проверить`.
- Если эти данные нужны для ответа, встроить их прямо в `Короткий ответ`.
- `Короткий ответ` — основной полный билет, а не короткая шпаргалка.
- Если в заголовке несколько вопросов, в `Коротком ответе` должны быть
  отдельные `####`-блоки по этим вопросам.
- Писать простыми словами: меньше канцелярита, меньше сложных оборотов.
- Ключевые мысли и термины выделять **жирным**.
- Длинные абзацы переписать в короткие абзацы или маркеры.
- Каждый маркер должен нести одну мысль.
- Блок `Схема / картинка` должен идти сразу после `Короткий ответ`.
- После картинки должен идти блок `Пояснение от ИИ простыми словами`.
- `Пояснение от ИИ простыми словами` объясняет содержание билета проще, но не
  добавляет новых фактов.
- Не писать мета-фразы вроде "в лекции", "в слайдах", "источники говорят",
  "лектор говорит".

## Что обновить

- `tickets/lecture-XX.md` — финальная версия билета.
- `tickets/lecture-XX.cram.md` — короткая версия для повторения.

## Финальный формат билета

```markdown
## Билет N. Точная формулировка вопроса

### Короткий ответ

#### Первая часть вопроса

- **...** — прямой ответ простыми словами.
- ...

#### Вторая часть вопроса

- **...** — прямой ответ простыми словами.
- ...

#### Пример

- **...** — пример из источников, если он помогает ответу.

### Схема / картинка

![Описание](../assets/lecture-XX/slide-001.png)

или `[подходящей картинки не найдено]`.

Если используется цельный слайд, картинка должна выглядеть как слайд на
`https://csa.edu.swampbuds.me/`: с заголовком, списками, подписями и layout-ом,
если они есть в браузерном рендере.

### Пояснение от ИИ простыми словами

Короткое понятное объяснение того, что написано в билете. Без новых фактов.
```

## Формат статуса в конце `tickets/lecture-XX.md`

```markdown
---

## Статус подготовки

- статус: `reviewed` / `needs-check`
- дата финализации: `YYYY-MM-DD`
- оставшиеся проверки:
  - ...
```

`reviewed` ставить только если все существенные утверждения подтверждены
источниками и нет критичных `[проверить]`.

`needs-check` ставить, если:

- транскрипт автоматический и есть сомнительные места;
- источник противоречив;
- вопрос из экзаменационного списка покрыт лекцией не полностью;
- локальные слайды не удалось уверенно сверить с правильным deck-ом;
- не хватает данных для уверенного ответа.

## Формат `tickets/lecture-XX.cram.md`

Краткая версия не должна содержать новых фактов. Она сжимает уже
финализированный билет.

```markdown
# Lecture XX — краткое повторение

## Главное

- **...**

## Определения

- **Термин** — определение.

## Формулы

$$
...
$$

## Нужно проверить

- `[проверить]` ...
- `[нет в источниках]` ...
```

Если формул нет, писать `[формул в источниках не найдено]`.

## Самопроверка

Перед завершением проверь:

- раздела `Подробный ответ` нет;
- старые служебные секции не возвращены по умолчанию;
- билет исправлен по ревью;
- краткая версия не содержит новых фактов;
- неподтверждённые утверждения удалены или помечены;
- `Короткий ответ` достаточно полный для устного ответа;
- если в заголовке несколько вопросов, каждый вопрос закрыт отдельным блоком;
- язык простой и понятный;
- ключевые мысли выделены **жирным**;
- длинные абзацы сокращены;
- картинки стоят сразу после `Короткий ответ`;
- есть `Пояснение от ИИ простыми словами`;
- пояснение простыми словами не добавляет новых фактов;
- в конце есть статус подготовки.
